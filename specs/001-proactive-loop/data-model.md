# Data Model: Identity-Aware Proactive Loop

**Feature**: `001-proactive-loop`
**Date**: 2026-03-06
**Storage**: Firebase Firestore

---

## Firestore Collection Hierarchy

```
users/{userId}                          ← Root user document
  ├── sessions/{sessionId}              ← Observation session
  │     └── interactions/{interactionId} ← Per-intervention log
  └── session_summaries/{summaryId}     ← End-of-session digest
```

---

## Entity: User (Core Identity Profile)

**Collection**: `users`
**Document ID**: Firebase Auth UID (e.g., `uid_haris_001`)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `user_id` | string | ✅ | Firebase Auth UID (mirrors document ID) |
| `name` | string | ✅ | Display name — "Haris" |
| `location` | string | ✅ | User's city — "Karachi, Pakistan" |
| `projects` | array\<ProjectEntry\> | ✅ | Active projects; see sub-schema below |
| `interests` | array\<string\> | ✅ | e.g., ["Rumi", "Turkish culture", "software craftsmanship"] |
| `last_updated` | timestamp | ✅ | Last time this document was written |

**ProjectEntry sub-schema** (map within `projects` array):

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | ✅ | Project name — e.g., "DoneKaro" |
| `status` | string | ✅ | Last known status — e.g., "Docker setup in progress" |
| `context` | string | ❌ | Optional freetext — e.g., "task manager app for local market" |

**Example document** (Haris's seed data):
```json
{
  "user_id": "uid_haris_001",
  "name": "Haris",
  "location": "Karachi, Pakistan",
  "projects": [
    { "name": "DoneKaro", "status": "Docker setup in progress", "context": "Task manager app" },
    { "name": "RehnumaAI", "status": "Auth flow complete", "context": "AI guide for students" }
  ],
  "interests": ["Rumi", "Turkish culture", "software craftsmanship"],
  "last_updated": "2026-03-06T00:00:00Z"
}
```

## Entity: Observation Session

**Collection**: `users/{userId}/sessions`
**Document ID**: auto-generated (e.g., `sess_20260306_001`)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `session_id` | string | ✅ | Mirrors document ID |
| `user_id` | string | ✅ | Firebase Auth UID |
| `started_at` | timestamp | ✅ | When observation was activated |
| `ended_at` | timestamp | ❌ | Set when session closes; null while active |
| `status` | string | ✅ | `active` \| `paused` \| `ended` |
| `interaction_ids` | array\<string\> | ✅ | List of interaction document IDs generated this session |
| `session_summary_id` | string | ❌ | Set after Auto-Summarizer completes; null until then |

**Status transitions**:
```
active → paused   (user clicks pause)
paused → active   (user re-enables observation)
active → ended    (user closes Mirr'at or 30min idle)
paused → ended    (30min idle while paused)
```

---

## Entity: Interaction Summary

**Collection**: `users/{userId}/sessions/{sessionId}/interactions`
**Document ID**: auto-generated (e.g., `int_20260306_001`)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `interaction_id` | string | ✅ | Mirrors document ID |
| `session_id` | string | ✅ | Parent session ID |
| `trigger_type` | string | ✅ | `A` (frustration) \| `B` (coding_block) |
| `triggered_at` | timestamp | ✅ | When the threshold was crossed |
| `intervention_text` | string | ✅ | Exact text of the intervention offered |
| `user_response` | string | ✅ | `accepted` \| `dismissed` \| `no_response` |
| `response_at` | timestamp | ❌ | When user responded; null for `no_response` |

**Immutability**: Documents are written once and never updated. If follow-up context is
needed, a new Interaction Summary is created.

**Privacy constraint**: This document MUST NOT contain raw video frames, audio, image data,
or any field derived from unprocessed biometric input. Only the inferred state label
(`trigger_type`) and text are stored.

## Entity: Session Summary

**Collection**: `users/{userId}/session_summaries`
**Document ID**: auto-generated (e.g., `sum_20260306`)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `summary_id` | string | ✅ | Mirrors document ID |
| `user_id` | string | ✅ | Firebase Auth UID |
| `session_id` | string | ✅ | The session this summary covers |
| `session_date` | string | ✅ | ISO date — "2026-03-06" (for display and ordering) |
| `summary_text` | string | ✅ | Exactly 2 sentences of plain text; generated by Auto-Summarizer |
| `session_duration_minutes` | number | ✅ | Duration from started_at to ended_at in minutes |
| `created_at` | timestamp | ✅ | When this summary was written to Firestore |

**Query pattern** (session-start loading):
```
users/{uid}/session_summaries
  .orderBy('created_at', 'desc')
  .limit(SESSION_SUMMARY_DEPTH)   // default: 3
```

**Retention**: No hard deletion policy for MVP. A rolling limit of 90 summaries (≈ 3 months)
may be enforced in a future version via a Cloud Function; not required for hackathon MVP.

---

## Entity: Trigger State (transient — never persisted)

This entity exists only in-memory during the observation loop. It is destroyed when the
session ends or the trigger fires. It MUST NOT be written to Firestore.

| Field | Type | Description |
|-------|------|-------------|
| `trigger_type` | string | `A` \| `B` |
| `first_detected_at` | datetime | When the current posture/state was first observed |
| `elapsed_seconds` | number | Seconds since first detection |
| `threshold_crossed` | bool | Whether the time threshold has been met |
| `last_intervention_at` | datetime \| None | Last time this trigger type fired (for cooldown) |

## Firestore Security Rules (outline)

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Only authenticated users can access their own data
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null
                         && request.auth.uid == userId
                         && request.auth.uid == "uid_haris_001";
    }

    // Deny all other access
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
```

**Note**: The hard-coded UID check (`== "uid_haris_001"`) is appropriate for the
single-user hackathon MVP. Remove this check to support multi-user scenarios.
